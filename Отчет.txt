Лабораторная 3, часть 1.

1. Постановка задачи. 
Написать функцию, которая будет отображать процентное соотношение размеров 
элементов относительно текущей папки с группировкой по элементам папки или по типам.
2. Предлагаемое решение
Для реализации данной задачи был применен паттерн стратегия.
Стратегия — это поведенческий паттерн проектирования, который определяет семейство схожих алгоритмов и помещает каждый 
из них в собственный класс, после чего алгоритмы можно взаимозаменять прямо во время исполнения программы.
Паттерн Стратегия предлагает определить семейство схожих 
алгоритмов, которые часто изменяются или расширяются, и вынести их 
в собственные классы, называемые стратегиями.

Этот паттерн применяется в случаях, когда нужно одну задачу решить разными методами\алгоритмами. Наша задача как раз подходит под такую - вычисление размера.
Вариации алгоритма - по папкам или по типам.
3. Шаги реализации. 
Для реализации данной задачи был разработан список классов, а именно:

CalculationStrategy - базовый класс для стратегий. Конкретные стратегии наследуются от этого класса. Содержит виртуальный метод calculate, который и осуществляет
подчет и группировку. На выходе объект структуры FileData. Структура FileData содержит в себе 2 поля - totalSize - вес всей папки и QMap<QString, double> - 
словарь с парами - расширение\имя и процент.
FileTypeStrategy - конкретная стратегия с группировкой по типам файлов.
FolderStrategy - конкретная стратегия с группировкой по папкам.
SizeCalculator - класс контекст, использующий стратегии.

4. Тестирование. (См main.cpp)
Были проработаны следующие ситуации: 
    //Тест 1 - пустая папка
    //Тест 2 - папка с файлами
    //Тест 3 - папка с вложениями, не пустая
    //Тест 4 - пустая папка с вложениями (не пустыми)
    //Тест 5 - Не пустая папка с вложениями(пустыми)
    //Тест 6 - много вложений, файлов.
    //тест 7 - один файл
    //тест 8 - неверный путь к директории (исключение)

Стоит отметить, что на данном этапе возвращается полный список файлов и дирректорий (без группировки в others). Такое решение позволит обработать данные в другом месте, например, для изменения минимального процента для отображения или
максимального числа файлов
